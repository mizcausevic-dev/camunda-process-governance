# Security Policy

This repository is a public demonstration surface using **synthetic process, task, SLA, and exception data only**.

## Scope

- No live Camunda tenant data, workflow payloads, customer records, or production audit trails should be committed here.
- The repo demonstrates process-governance and review-posture patterns, not a production orchestration control plane.
- Any future embedded or hosted tier must use least-privilege integrations, read-only evidence paths where possible, and signed execution packets.

## Reporting

If you find a security issue in the code or docs, open a private report through GitHub security advisories or contact the maintainer directly.
